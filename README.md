## Step 4: 提出修改建议

_很棒！你已经成功完成了一次审查 :sparkles:_

接下来，我们来学习 GitHub 里一个非常实用的功能 —— **提出修改建议（Suggest changes）**。

**什么是 _suggest changes_?**: 

当你在审查代码时，可能会发现一些小问题，比如代码不规范，或者某行逻辑可以更简洁。
这时你就可以直接在评论中**给出修改建议**。
更方便的是，PR 的作者可以**一键采纳你的建议**并提交，无需手动修改代码。

这个功能让代码审查不仅是讨论，更是一次高效的协作。

### :keyboard: 实操环节

1. 打开 Pull Request 页面，点击 **Files changed**。
2. 找到文件 `index.html` 的修改内容。
3. 把鼠标移动到代码左侧的行号旁边。
4. 点击出现的蓝色加号图标。
5. 在评论输入框中，点击 **Add a suggestion（添加建议）** 按钮。 <br>
   ![add-a-suggestion-button](https://user-images.githubusercontent.com/97056108/184449714-61e8ee51-824a-48c1-9436-2dfd67f2c070.png)
6. 修改建议内容，写上你认为更合适的代码或文本。
7. 点击 **Add a single comment（添加单条评论）** 提交建议。
8. 等待约 20 秒后刷新此页面。
   [GitHub Actions](https://docs.github.com/en/actions) 会检测到你的操作并进入下一步。
